## 共通の語彙

ブログ記事とドキュメントページは `src/config/tag-vocabulary.ts` のひとつの語彙を
共有しています。これにより名前空間が一貫し、`content` タグの付いた記事は同じく
`content` タグの付いたドキュメントと並びます。タグ監査スクリプトは両方の
コレクションをまたいで未知の id を一括で報告してくれます。

## ガバナンスのモード

タグ検証の厳しさはふたつの設定で決まります。`tagVocabulary` は語彙ファイルを
そもそも参照するかどうか、`tagGovernance` は参照する場合の強制レベル
(`off` / `warn` / `strict`) を決めます。デフォルトは `tagVocabulary: true` と
`tagGovernance: "warn"` の組で、開発中の素早いイテレーションを許しつつ、
誤字は監査の出力で気づけるようになっています。

## 新しいタグを追加する

語彙に存在しないタグを使いたいときは、記事に直接書くのではなく
`tag-vocabulary.ts` にエントリを追加します。スキーマは次の形です。

```ts
{
  id: "performance",
  description: "Performance tuning, profiling, build speed.",
  group: "topic",
}
```

古い表記は `aliases` で吸収し、運用を止めたいタグは `deprecated` を使うと
既存ページを壊さずに段階的に廃止できます。

## なぜこの記事は手書き抜粋を使うのか

この記事のフロントマターには `excerpt:` フィールドが書かれています。ブログ
インデックスはその値を抜粋として表示し、`<!-- more -->` マーカーよりも優先
されます。動詞から始めたいときや、長い前置きを抜粋に出したくないときなど、
要約文を本文の冒頭と別の言い回しにしたい場面で重宝します。
