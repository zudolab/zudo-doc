## 概要

カラースキームプレビューは、サイトヘッダーにドロップダウンセレクターを追加し、利用可能なすべてのカラースキームをランタイムで切り替えられるようにします。設定ファイルを変更せずに、さまざまなテーマをプレビューするのに便利です。

## 仕組み

有効にすると、ヘッダー（デスクトップ・モバイル両方）に `<select>` ドロップダウンが表示されます。`src/config/color-schemes.ts` で定義されたすべてのカラースキームがリストされます。スキームを選択すると:

- スキームの CSS カスタムプロパティがページに即座に適用されます
- 選択は `localStorage` に保存され、ページリロードやナビゲーションをまたいで維持されます
- ソースファイルは変更されません — 変更はクライアントサイドのみです

## カラースキームプレビューの有効化

### create-zudo-doc での設定

カラースキームプレビューはデフォルトで無効です。プロジェクト作成時に有効にするには:

```bash
pnpm create zudo-doc my-docs --color-scheme-preview
```

### 既存プロジェクトでの設定

カラースキームプレビューは `src/config/settings.ts` の `colorMode` 設定で制御されます。ヘッダーはこの設定に基づいて 2 つのコントロールのいずれかをレンダリングします:

- `colorMode` が設定されている場合（`defaultMode`、`lightScheme`、`darkScheme` を持つオブジェクト）、ヘッダーには**ライト/ダーク切り替え**ボタンが表示されます
- `colorMode` が `false` の場合、ヘッダーには**カラースキームピッカー**のドロップダウンが表示されます

既存プロジェクトでカラースキームプレビューを有効にするには、`colorMode` を `false` に設定します:

```ts title="src/config/settings.ts"
export const settings = {
  colorScheme: "Dracula",
  colorMode: false,
  // ...
};
```

<Note>
`colorMode` を `false` に設定すると、ライト/ダーク切り替えが無効になります。サイトは単一カラースキームモードを使用し、ピッカーで利用可能なすべてのスキームを閲覧できます。
</Note>

## ライト/ダークモードとの関係

カラースキームピッカーとライト/ダークテーマトグルは、ヘッダー上で排他的に表示されます:

- **ライト/ダークモード**（`colorMode` がオブジェクト）: 事前設定されたライトとダークのスキームペアを切り替えます。`ThemeToggle` コンポーネントが表示されます。
- **単一スキームモード**（`colorMode` が `false`）: `ColorSchemePicker` ドロップダウンですべてのスキームから選択できます。

ライト/ダーク切り替えとスキーム閲覧の両方が必要な場合は、ヘッダーコンポーネントのカスタマイズが必要です。

## 利用可能なカラースキーム

カラースキームは `src/config/color-schemes.ts` で定義されています。各スキームは 16 色パレット、背景/前景色、選択色、Shiki コードハイライトテーマ、およびオプションのセマンティックカラーオーバーライドを指定します。

`create-zudo-doc` CLI には最大 40 の組み込みスキームが含まれています。`color-schemes.ts` の `colorSchemes` オブジェクトにエントリを追加することで、カスタムスキームを追加できます。

<Tip>
カラースキームプレビューは、開発中に最終的なスキームを決定する前に、さまざまなテーマでコンテンツの見た目をすばやく確認するのに特に便利です。
</Tip>
