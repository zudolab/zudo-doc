## 概要

カラースキームプレビューは、サイトヘッダーにドロップダウンセレクターを追加し、利用可能なすべてのカラースキームをランタイムで切り替えられるようにします。設定ファイルを変更せずにさまざまなテーマをプレビューするのに便利です。

## 仕組み

有効にすると、ヘッダー（デスクトップとモバイルの両方）に `<select>` ドロップダウンが表示されます。`src/config/color-schemes.ts` で定義されたすべてのカラースキームが一覧表示されます。スキームを選択すると:

- スキームの CSS カスタムプロパティがページに即座に適用されます
- 選択は `localStorage` に保存され、ページのリロードやナビゲーションをまたいで維持されます
- ソースファイルは変更されません — 変更はクライアントサイドのみです

## カラースキームプレビューの有効化

### create-zudo-doc の場合

カラースキームプレビューはデフォルトで無効です。プロジェクト作成時に有効にするには:

```bash
pnpm create zudo-doc my-docs --color-scheme-preview
```

### 既存プロジェクトの場合

カラースキームプレビューは `src/config/settings.ts` の `colorMode` 設定で制御されます。ヘッダーはこの設定に基づいて2つのコントロールのいずれかを表示します:

- `colorMode` が設定されている場合（`defaultMode`、`lightScheme`、`darkScheme` を持つオブジェクト）、ヘッダーには **ライト/ダークトグル** ボタンが表示されます
- `colorMode` が `false` の場合、ヘッダーには **カラースキームピッカー** ドロップダウンが表示されます

既存プロジェクトでカラースキームプレビューを有効にするには、`colorMode` を `false` に設定します:

```ts title="src/config/settings.ts"
export const settings = {
  colorScheme: "Dracula",
  colorMode: false,
  // ...
};
```

<Note>

`colorMode` を `false` に設定すると、ライト/ダークトグルは無効になります。サイトは単一カラースキームモードを使用し、ピッカーでユーザーがすべての利用可能なスキームを閲覧できます。

</Note>

## ライト/ダークモードとの関係

カラースキームピッカーとライト/ダークテーマトグルは、ヘッダーで相互に排他的です:

- **ライト/ダークモード**（`colorMode` がオブジェクト）: ユーザーは事前設定されたライトとダークのスキームペアを切り替えます。`ThemeToggle` コンポーネントが表示されます。
- **シングルスキームモード**（`colorMode` が `false`）: ユーザーは `ColorSchemePicker` ドロップダウンですべての利用可能なスキームから選択します。

ライト/ダークの切り替えとスキームの閲覧の両方が必要な場合は、ヘッダーコンポーネントをカスタマイズする必要があります。

## 利用可能なカラースキーム

カラースキームは `src/config/color-schemes.ts` で定義されています。各スキームは16色パレット、背景色/前景色、選択色、Shiki コードハイライトテーマ、およびオプションのセマンティックカラーオーバーライドを指定します。

`create-zudo-doc` CLI には最大40の組み込みスキームが含まれています。`color-schemes.ts` の `colorSchemes` オブジェクトにエントリを追加することでカスタムスキームを追加できます。

<Tip>

カラースキームプレビューは、最終的なスキームを決定する前に、さまざまなテーマでコンテンツがどのように表示されるかをすばやく評価するのに特に便利です。

</Tip>
