# S5 Footer, Footer Taglist & Body Foot Util Area вҖ” Audit

**Build base:** branch `zfb-feature-audit/s5-footer`, built with `SKIP_DOC_HISTORY=1 pnpm build` on 2026-05-04.  
**Pages inspected (default config):** `dist/docs/guides/footer/index.html`, `dist/ja/docs/guides/footer/index.html`, `dist/docs/guides/body-foot-util-area/index.html`, `dist/ja/docs/guides/body-foot-util-area/index.html`, `dist/docs/getting-started/index.html`.  
**Sandbox builds:** taglist `groupBy: "group"`, taglist `groupBy: "flat"`, `footer: false`, `bodyFootUtilArea: false` вҖ” all verified via separate builds, `git restore` applied after each.  
**Sources read:** `packages/zudo-doc-v2/src/footer/footer.tsx`, `footer/types.ts`, `pages/lib/_footer-with-defaults.tsx`, `pages/lib/_doc-history-area.tsx`, `packages/zudo-doc-v2/src/body-foot-util/body-foot-util-area.tsx`, `body-foot-util/edit-link.tsx`, `src/config/settings.ts`, `src/config/settings-types.ts`, `src/utils/github.ts`, `src/config/i18n.ts`.

**Conclusion:** Footer link columns, copyright HTML, locale labels, and taglist modes all work correctly. One functional bug found: `bodyFootUtilArea.docHistory` is defined in the type and documented but never read at runtime вҖ” only the top-level `settings.docHistory` gates the history island. Setting `bodyFootUtilArea: false` does NOT suppress the history island; the doc claims it does. Additionally, `EditLink` (for the `editUrl` "Edit this page" feature) is exported from v2 but never wired into the host's `_doc-history-area.tsx`.
